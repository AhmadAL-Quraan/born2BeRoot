## How many commands executed with `sudo` on the system 

* The number of commands executed with the sudo program:
```bash
journalctl _COMM=<command> | grep COMMAND | wc -l 
```

* What `_COMM` do: **filters journal** logs to show only entries generated by a specific command (process name).

>[!note] Only programs that actually write to the system journal will appear.  
Many simple commands like `ls`, `cat`, etc., do not log anything by default, so they won’t show up in `journalctl`.

## Get the network interface and IP address (public / private)
* How to know the private IP address and the network interface (MAC) -> If by ethernet or wifi .

`ip route get 1.1.1.1` -> give the route table of the current connection. It won't send packets to this IP, it will just show the rules of who and where to send those packets.

- `1.1.1.1` = a **DNS server**    
- It’s used to translate domain names → IP addresses  
    (like `google.com` → `142.250.x.x`)


###  Why we use it in commands

When you run:
`ip route get 1.1.1.1`
👉 You’re basically asking:
> “If I try to reach the internet, which interface and IP will my system use?”


### Why `1.1.1.1` specifically?

-  Always online
-  Very fast and reliable
-  Public (reachable from anywhere)
-  Doesn’t depend on your network
So it's perfect as a **test destination**

## Redirect all the errors from the script into /dev/null

```bash
exec 2>/dev/null
```


-> `exec command`: Replace current program (bash shell) with another program (command) on the same process. Original program (bash) will die (gone).
-> `exec > file` → redirects output globally
-> `exec 2>/dev/null` → hides all errors globally
- No new process is created




## See the current users who using the system right now and their terminal 


* Use the command `who` 

### Different between `w` and `who`
#### `who`

- Reads from **`/var/run/utmp`**
- Shows only **real login sessions**
    - TTY logins
    - SSH sessions
    - display manager sessions

👉 So it only lists users that **officially logged into the system**
![[Pasted image 20260320235757.png]]

####  `w`
- Uses **process information (`/proc`)**
- Shows **any active session with a controlling terminal or user context**

👉 This includes:
- Real users (like you)
- **System/service users** (like `exam` in your case) --> Services that acts as users.

![[Pasted image 20260320235743.png]]

>[!note] So `w` and `who` could show different users, because both of them read from different files on the system, `w` shows also any service that acts as a user, unlike `who` which shows real users only. `w` Reads from the `/proc`.
